This site has a blog now — a place to put down thoughts that don't fit in a
commit message or a project README.

It works the same way as the rest of the site:

- Each post is a plain **Markdown** file in `blog/posts/`
- `posts.json` is the index — one entry per post
- A small vanilla JavaScript renderer turns the Markdown into HTML in the browser

No frameworks, no build step, no comments section. Just writing.

## What to expect

Probably notes on whatever I'm building at the time — backend work, data,
deployment — and the occasional thought that has nothing to do with software
at all. No schedule.
